# 二期增强说明

## 设计动机

第一期版本已经完成了检测、样本入库、批量评测、误报漏报归因和基础报告生成，但它更偏“安全能力验证”。第二期增强的目标，是把项目继续推进成一个更适合创赛展示和答辩讲解的数据分析与策略优化平台。

这轮增强重点解决三个问题：

- 样本是否足够干净、可用、可复核。
- 规则是否能从“有命中”进一步走向“可运营、可统计、可解释”。
- 误报漏报是否能沉淀为案例资产，而不是只停留在一次评测输出里。

## 主要改动

### 1. 样本质量审计与复核队列

- 为 `Sample` 扩展 `label_confidence`、`duplicate_group_id`、`boundary_sample_flag`、`needs_review`、`review_comment` 字段。
- 新增 `scripts/audit_samples.py`，自动识别重复样本、高相似样本、标签缺失、标签冲突和边界样本。
- 审计结果写入 `docs/reports/sample_audit_*.md`。
- Streamlit 增加“样本复核”页面，支持按 `needs_review`、`boundary_sample_flag`、`attack_type`、`scenario` 筛选。

### 2. 规则版本与规则效果分析

- 为规则补充 `rule_version`、`created_by`、`updated_at`、`change_note` 字段。
- 新增 `scripts/analyze_rule_effectiveness.py`。
- 支持查看规则命中次数、样本类型分布、攻击类型分布、拦截贡献、review 贡献、误报数和漏报修复潜力。
- Streamlit 增加“规则运营分析”页面。

### 3. 误报漏报案例中心

- 新增 `CaseRecord` 数据结构，保存典型误报和漏报案例。
- 新增 `scripts/build_casebook.py`，可从评测结果中自动抽取典型案例。
- 案例文档写入 `docs/cases/`。
- Streamlit 增加“案例中心”页面，支持按 `case_type`、`attack_type`、`root_cause`、`status` 查看。

### 4. 多策略评测对比

- 扩展策略配置，支持 `strategy_version`、`rule_selection`、`output_filter_threshold`。
- 新增 `scripts/compare_strategies.py`，支持多个 `evaluation run` 横向比较。
- 支持总体指标、按攻击类型聚合、按样本类型聚合和阈值扫描结果展示。
- Streamlit 增加“策略对比”页面。

### 5. 周报、复盘与运营材料

- 新增 `scripts/generate_weekly_report.py`。
- 新增 `scripts/generate_postmortem.py`。
- 周报输出到 `docs/weekly_reports/`，复盘输出到 `docs/postmortems/`。
- 前端“报告中心”可统一浏览评测报告、案例文档、周报和复盘。

## 使用方式

初始化与启动：

```bash
make init
make run
make ui
```

第二轮增强常用命令：

```bash
make audit
make rule-report
make cases
make compare
make weekly
make postmortem
```

如果希望通过 API 触发：

- `GET /samples/review_queue`
- `GET /rules/effectiveness`
- `GET /cases`
- `POST /cases/build`
- `POST /reports/weekly`
- `POST /reports/postmortem`
- `GET /evaluations/compare`

## 为什么文档材料落在文件系统

周报、复盘和案例分析的主要消费者是人，而不是在线事务型查询。对这类材料来说，Markdown 文件更适合：

- 用 Git 跟踪变更历史。
- 直接作为创赛答辩材料。
- 和评测报告、SOP 放在同一目录体系中。

因此本项目把运营材料放在 `docs/weekly_reports/`、`docs/postmortems/`、`docs/cases/`，数据库只保留与案例检索相关的结构化索引。

## 创赛答辩亮点

这轮增强让项目从“能拦截”升级到“能运营、能分析、能复盘”，更适合在创赛答辩里展示完整的方法论和工程闭环：

- 能建设样本治理机制，而不是只导入样本。
- 能分析规则效果，支持规则级优化决策。
- 能沉淀误报漏报案例，形成知识资产。
- 能输出周报和复盘材料，体现日常运营工作方法。
- 能用前端看板展示策略变化、样本变化和案例变化，适合现场答辩讲解。
