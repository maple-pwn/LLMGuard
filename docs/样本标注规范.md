# 样本标注规范

## 目标

本规范用于统一建设 LLM Firewall 样本库，保证攻击样本、对抗样本、白样本在后续规则评测、阈值调优和误报漏报分析中具有可比性。

## 基础字段

- `text`：主输入文本，默认表示用户输入。
- `sample_type`：`attack` / `adversarial` / `benign`。
- `attack_category`：攻击大类，例如 `direct_prompt_injection`、`indirect_prompt_injection`、`jailbreak`、`sensitive_info_exfiltration`、`role_override`、`tool_misuse_attempt`。
- `attack_subtype`：更细粒度的子类，例如 `ignore_previous`、`output_side_leak`。
- `risk_level`：`low` / `medium` / `high` / `critical`。
- `source`：样本来源，建议记录为 `manual_seed`、`redteam`、`production_case`、`public_dataset_adapted`。
- `tags`：场景标签与补充信息，例如 `办公助手`、`知识库问答`、`迷惑性白样本`。
- `expected_result`：期望结果，取值 `allow` / `review` / `block`。
- `reviewer`：标注人或复核人。
- `review_status`：`pending` / `approved` / `rejected`。
- `retrieved_context`：RAG 场景检索文本。
- `model_output`：输出侧过滤样本对应的模型响应。

## 标注规则

1. 如果风险来自用户输入本身，`text` 中直接保留攻击文本，`retrieved_context` 为空。
2. 如果风险来自外部知识库或网页检索结果，`text` 应保持正常业务问题，恶意指令写入 `retrieved_context`。
3. 如果风险发生在模型输出阶段，`text` 可以是正常问题，泄露内容或违规内容写入 `model_output`。
4. `expected_result` 以运营策略为准：
   - 明确越权、泄露、工具滥用请求，一般标注为 `block`。
   - 具有较强迷惑性、可能需要人工判断的对抗样本，可标注为 `review`。
   - 正常业务问答与知识解释，标注为 `allow`。
5. 白样本中至少保留一部分“迷惑性白样本”，例如讨论系统提示词概念、解释 token 风险、讲解 jailbreak 定义等，用于验证误报分析能力。

## 复核建议

1. 每条新增攻击样本至少做一次复核，避免只凭主观判断打标签。
2. 对误报率较高的规则相关样本增加 `迷惑性白样本` 标签，便于后续聚类分析。
3. 对线上真实案例样本增加来源与脱敏说明，避免泄露内部数据。
