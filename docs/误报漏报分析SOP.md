# 误报漏报分析 SOP

## 适用范围

适用于本项目中规则引擎、分类器、输出侧过滤三类能力的误报与漏报复盘。

## 输入材料

- 最新一轮评测报告
- `EvaluationRun.metrics`
- `DetectionResult` 明细
- 对应样本原文、上下文与标签

## 误报分析步骤

1. 筛选 `expected_result=allow` 且 `decision!=allow` 的样本。
2. 按 `triggered_rules` 聚合，观察是否集中在少数高频规则。
3. 判断误报类型：
   - 规则命中明显但语义是科普/培训/翻译，标注为 `白样本与攻击模板局部重叠`。
   - 单条规则覆盖范围过宽，缺少语义边界，标注为 `规则范围过宽`。
   - 仅分类器高分触发且规则未命中，标注为 `分类器泛化不足`。
   - 仅输出侧过滤触发且输出为安全解释文本，标注为 `输出侧过滤阈值偏低`。
4. 给出修复动作：
   - 增加否定语义规则或白名单上下文。
   - 调低规则权重，改为人工复核。
   - 补充同类白样本重新训练分类器。

## 漏报分析步骤

1. 筛选 `expected_result!=allow` 且 `decision=allow` 的样本。
2. 判断是否完全无规则命中：
   - 无规则命中且分类器低分，多为 `规则覆盖不足`。
   - RAG 场景只看用户输入没看上下文，标注为 `上下文信息缺失`。
   - 命中弱规则但未过阈值，可视为 `分类器泛化不足` 或阈值偏高。
3. 检查攻击类别与命中规则是否错位，例如角色劫持文案未被角色类规则捕获。
4. 给出修复动作：
   - 新增规则模板或补充正则别名。
   - 扩充对抗样本，提高分类器对伪装语气和业务包装文本的识别能力。
   - 对特定场景单独收紧阈值，例如 `knowledge_base_qa` 的间接注入场景。

## 输出要求

每轮分析至少产出：

- 误报 Top 5 规则
- 漏报 Top 5 攻击子类
- 本轮新增样本建议
- 本轮策略调整建议
